--- !set variable value ----

allowed_clients - the list of clients allowed to connect to the hub (basing on TAG description eg. DC++ tag is "++")
note: '!set allowed_clients ' (one space after allowed_clients) disables client checking at all (all clients are allowed to connect)
autofav - a frequency of sending auto-fav message (stored in 'autofav' file, which can be set with !set autofavmsg command) to all unregistered users, it is sending every autofav * hub_timer (default: 15 minutes) minutes
note: 'set autofav 0' disables sending of 'autofavmsg' message at all
ban_overrides_allow - stresses if ban entry is stronger that allow entry
0 - allow entry is stronger than ban entry [default]
1 - ban entry is stronger that allow entry
banmsg - message send to banned user while trying to log in
botnick - nickname for main bot sending all messages to users (default: Security)
check_class - stresses which classes will be checked for information stored in TAG
0 - REGULAR [default]
1 - REGULAR, REG
2 - REGULAR, REG, VIP
3 - REGULAR, REG, VIP, KVIP
4 - REGULAR, REG, VIP, KVIP, OP
config_on_priv - sending !getconfig (hub settings) output to admins as a private message from botnick instead of main chat message
0 - off [default]
1 - on
connlimit - the lowest limit of upload bradwith user can set without being kicked (default 10 kB/sec)
note: works with tag B:, L: and NetLimiter connection type
count_reghubs - adds counting hubs where user is registered to all hubs (normal+registered, without opped -> H:n/r/o) while checking for maxhubs
crypt_enable - crypting passwords in 'reglist' file
0 - none, human-readable passwords [default]
1 - password will be crypted
warning: after switching that all old password will not work! it's recommended to set this at the first running of hub
cut_reg_msg - cuts also too long main chat messages of REGISTRED users to !set to cut_string_to characters
cut_string_to - maximum number of characters that can be send by unregistered user to main chat(longer messages will be cut)
default_pass - if set, unregistered users will be prompted for that password (pass) while logging in
default_prefixes - if set, unregistered users' nicknames will have to contain one of the prefixes you can type here (separated by spaces)
deny_passive - restrictions for the users with M:P and M:5 in TAG
0 - no restrictions [default]
1 - kick
2 - kick if nickname doesn't contain 'psv' string
note: checking depends on check_class
deny_pm_http - kicking users using http:// string in private messages and reporting it to ops/admins
0 - off [default]
1 - on, kicking for any pm http://
2 - on, kicking after 5 pm messages from one user (good for adultfinder spammers)
note: checking depends on check_class
deny_proxy - kicking users for proxy connections (M:5)
0 - off [default]
1 - on
note: checking depends on check_class
expiredrls - time of expiration entries added to release database (in days)
note: after changing that, expration time of old entries isn't changed
note: setting to 0 means no expiration
expiredseen - time of expiration entries stored in seen database (in days)
note: after changing that, expration time of old entries isn't changed
expiredtime - time of unused account expiration (in days)
note: resetted every time registered user is logging in
note: after changing that, expration time of old entries isn't changed
note: setting to 0 means no expiration
expiredurl - time of URL expiration (in days)
note: after changing that, expration time of old entries isn't changed
note: setting to 0 means no expiration, no recommended
expirewarn - time of warning expiration (in days)
note: after changing that, expration time of old entries isn't changed
note: setting to 0 means no expiration, no recommended
freeaccess - temporary unrestricted access for unregistered users when registered_only is set to 1
free access period is in x-y format, where x is the hour and y is the period eg. 14-2 means free access from 14:00 to 16:00
note: works probably only with 24-hour based clock on server
forbchars - first letter of unregistered user's nickname cannot be any of the character set there (default: !#)
gag_on_minshare - gagging users who don't have enough data shared from searching and downloading instead of kicking or redirecting
help_on_priv - sending !help output as a private message from botnick instead of main chat message
0 - off [default]
1 - on
hide_opshare - hiding sharesize of ops/admins while logging in and exclude them for searches
0 - off [default]
1 - on
note: files from that users can be downloaded after downloading user's filelist
hub_full_mess - message send to users who can not enter the hub because of too many connected users
note: see max_users
hublist_upload - period of sending information to public_hub_host
0 - disabled
note: number*15 minutes eg. 1=15 minutes, 2=30 minutes etc.
hublistnick - nickname for hublist.org pinger (except its from all checking)
hublistpass - password for hublist.org pinger
warning: the same password has to be set first on hublist.org (in hub owner mode), otherwise access for pinger will be denied
hublistsite - if pinger is registered on hublist.org and if hublistpass is set, you can set also linkt to your hub's site on hublist.org so users will be informed when pinger is checking the hub
ipgag_type - gag type for ipgagged (with !gagip command) users
0 - gag from chatting [default]
1 - gag from searching and downloading
2 - gag from chatting, searching and downloading
hidetags - hide sending $MyINFO's TAGs, description and email to unregistered users
0 - off [default]

note: the purpose of hiding TAGs is to save hub resources
hub_description - short description of your hub, visible on hublists
hub_hostname - IP/hostname of server with hub (without port), hub set it by default while first running, maximum 100 characters
hub_name - just the name of your hub, maximum 60 characters
kick_bantime - time of tempban after kicking user from nicklist if no _ban_xx in reason was added
kvipprefix - nickname prefix for REGISTERED/KVIP users

note: gives 1 minute IP tempban

note: only unregistered users are counted
max_warnings - number of warnings allowed before being kicked

maxhubs - maximum number of hubs user is allowed to be connected to

note: see count_reghubs
maxslots - maximum number of upload slots user can open

note: can be set easier with !set min_vip_share x GB, !set min_share x MB
min_kvip_share - minimum share for KVIP users in bytes

mindcppver - minimum version of DC++ allowed to enter the hub

--- !block/!unblock main/pm/download/search ----

!block main - blocks sending messages to main chat for non-opped users (useful while flooding attacks)
!block pm - blocks sending private messages for non-opped users
!block search - blocks searching for non-opped users
!block download - blocks downloading for non-opped users
!unblock main - unblocks sending messages to main chat for non-opped users
!unblock pm - unblocks sending private messages for non-opped users
!unblock search - unblocks searching for non-opped users
!unblock download - unblocks downloading for non-opped users
